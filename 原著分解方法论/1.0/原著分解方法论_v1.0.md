# 原著分解方法论 v1.0

> 版本：1.0
> 基于案例：《孔乙己》（鲁迅，短篇小说）
> 创建日期：2025-12-15
> 目的：将线性叙述的文字转化为可编辑的故事元素，类似于将小说改编为游戏剧本

---

## 核心理念

**分解的本质**：不是「拆解文字」，而是「提取故事的可操作元素」。

我们要回答的问题是：如果一个用户想创建一个角色进入这个故事，他需要知道什么？系统需要理解什么？

---

## 六大分解维度

### 1. 故事骨架（Plot Structure）

**目标**：提取「发生了什么事」以及「为什么会发生」

**提取内容**：
- **事件链**：A → B → C 的因果序列
- **因果关系**：每个事件发生的原因
- **转折点**：故事方向改变的关键节点

**输出格式**：
```
【事件链】
事件A（原因） → 事件B（原因） → 事件C（原因） → ...

【转折点清单】
| 转折编号 | 事件描述 | 转折意义 |
|---------|---------|---------|
| T1 | ... | 从X变成Y |
```

**《孔乙己》案例**：
```
白骨精想吃唐僧 → 变化成村姑接近 → 悟空识破打死 → 唐僧误解责骂 → ...
转折点：偷到丁举人家被打断腿（从「可笑的落魄」→「悲惨的毁灭」）
```

---

### 2. 人物关系网（Character Dynamics）

**目标**：提取人物之间的动态关系，而非静态身份

**提取内容**：
- **人物清单**：身份、与主角的关系
- **张力网**：谁和谁之间有冲突/渴望/依赖
- **关系变化**：关系在故事中如何演变

**输出格式**：
```
【人物清单】
| 人物 | 身份 | 与主角关系 | 功能 |
|-----|-----|----------|-----|

【张力网】
人物A ←(冲突/渴望/依赖)→ 人物B

【关系变化曲线】
人物A与人物B：状态1 → 事件X → 状态2
```

**关键洞察**：
- 重要的不是「谁是谁」，而是「谁和谁之间有什么张力」
- 关系是动态的，需要追踪变化

---

### 3. 情感张力（Emotional Tension）

**目标**：提取作者设计的读者情感体验

**提取内容**：
- **情感递进**：读者在不同阶段的情感状态
- **表层 vs 深层**：表面看到的 vs 真正感受到的
- **情感转折**：读者情感发生转变的节点

**输出格式**：
```
【情感曲线】
| 阶段 | 表面情绪 | 深层情绪 | 触发元素 |
|-----|---------|---------|---------|
```

**《孔乙己》案例**：
| 阶段 | 表面情绪 | 深层情绪 |
|-----|---------|---------|
| 开场 | 滑稽、好笑 | 隐隐的不适 |
| "窃书不算偷" | 可笑的狡辩 | 读书人最后的尊严 |
| 断腿后出场 | 震惊 | 悲凉、愤怒 |
| 结尾 | 平静 | 深沉的悲哀与控诉 |

**关键洞察**：
- 情感曲线是隐藏的主线
- 伟大的作品往往让读者的情感发生转变（如从笑到哭）

---

### 4. 世界规则（World Rules）

**目标**：提取故事中隐含的规则系统

**提取内容**：
- **社会规则**：阶层、身份、权力的运作方式
- **物理规则**：这个世界的物理法则（如魔法系统）
- **道德规则**：什么被认为是对/错的（可能是扭曲的）

**输出格式**：
```
【规则清单】
| 规则类型 | 规则内容 | 体现细节 |
|---------|---------|---------|
```

**《孔乙己》案例**：
| 规则 | 内容 | 细节 |
|-----|-----|-----|
| 阶层规则 | 长衫=读书人=坐着喝，短衣=劳动者=站着喝 | 酒店格局描写 |
| 科举规则 | 没有进学=失败者 | "终于没有进学" |
| 暴力规则 | 有权者可以肆意施暴 | 丁举人打人无人过问 |

**关键洞察**：
- 世界规则往往是隐含的，需要从细节中提炼
- 规则定义了「什么是可能的」和「什么是正常的」

---

### 5. 可改写空间（Narrative Flexibility）

**目标**：识别哪些元素可以改变，哪些是故事DNA

**提取内容**：
- **可改变元素**：不影响故事核心的部分
- **不可改变元素**：故事的DNA，改了就不是这个故事了
- **用户角色切入点**：新角色可以从哪里进入

**输出格式**：
```
【可改变】
- ...
- ...

【不可改变（故事DNA）】
- ...
- ...

【用户角色切入点】
- 可能的身份：...
- 可能的介入时机：...
```

**《孔乙己》案例**：
- **可改变**：有人为孔乙己说话、"我"认真学了茴字、有人帮助断腿的孔乙己
- **不可改变**：孔乙己的科举失败、社会的冷漠本质、阶层分化
- **切入点**：酒店里的另一个顾客、"我"的朋友、孔乙己抄书时认识的人

---

### 6. 场景清单（Scene Breakdown）

**目标**：将故事拆分为可管理的最小叙事单位

**提取内容**：
- **场景编号**：唯一标识
- **场景名称**：简短描述性名称
- **地点**：发生在哪里
- **人物**：谁参与了
- **核心事件**：这个场景发生了什么

**输出格式**：
```
| 场景编号 | 场景名 | 地点 | 人物 | 核心事件 |
|---------|-------|-----|-----|---------|
| S1 | ... | ... | ... | ... |
```

**关键洞察**：
- 最小分解单位应该是「场景」而非「章节」
- 场景是「时间+地点+人物+事件」的组合

---

## 分解流程

```
1. 通读全文，理解整体
      ↓
2. 识别场景边界，拆分场景清单
      ↓
3. 提取事件链和转折点（故事骨架）
      ↓
4. 绘制人物关系张力网
      ↓
5. 标注情感曲线
      ↓
6. 提炼隐含的世界规则
      ↓
7. 判断可改写空间
      ↓
8. 输出结构化分解结果
```
